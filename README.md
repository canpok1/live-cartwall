# Tab Audio Console

画面共有しているタブの**中で**BGM・効果音を再生する操作卓。
再生を操作するパネルは**別ウィンドウ**なので、共有画面には映りません。

タブ音声を共有できるツールであれば何でも使えます（Google Meet / Zoom / Teams / OBS など）。
仮想オーディオデバイス（BlackHole、VB-CABLE 等）のインストールは不要です。

---

## 仕組み

```
[操作パネル（別ウィンドウ）]  ← 操作する人が触る。共有には映らない
          │  「Q2 を鳴らせ」
          ▼
[Service Worker] ──► [対象タブ内の音声エンジン]
                          │  Web Audio API で再生
                          ▼
                     タブの音声 ──► 「タブの音声も共有」──► 参加者
```

音は対象タブの音声そのものになるため、マイクを経由しません。
Web会議ツールのノイズキャンセルに削られず、音楽がクリアに届きます。

---

## インストール

1. このフォルダを任意の場所に置く（例: `~/tab-audio-console`）
2. Chrome で `chrome://extensions` を開く
3. 右上の **デベロッパーモード** をON
4. **パッケージ化されていない拡張機能を読み込む** → このフォルダを選択
5. ツールバーの拡張アイコンをクリック → 操作パネルが別ウィンドウで開く

---

## 事前準備

1. 操作パネルを開く
2. **音源を追加** に音声ファイルをドロップ
   - 30秒以上のファイルは自動で **BGM**（ループ・フェード付き・音量控えめ）
   - 30秒未満は自動で **効果音**（一発・重ねがけ可）
   - 判定は BGM / 効果音 のトグルでいつでも変更可
3. 各キューの **VOL** を調整。BGMは35%前後が目安（話し声を邪魔しない）

音源はブラウザに保存されるので、一度読み込めばPCを再起動しても残ります。

---

## 使い方

1. 共有したいページ（スライドなど）を開き、操作パネルの「出力先タブ」でそのタブを選んで **このタブに接続**
2. **対象タブを一度クリック**（ブラウザの自動再生制限を解除する。パネル上部が「待機中」になればOK）
3. Web会議ツールで **画面を共有 → タブ** を選択し、**「タブの音声も共有」をON**
4. 操作パネルの **再生** ボタンでキュー出し

---

## 事故を防ぐための注意

- **共有時は必ず「タブ」を選ぶこと。** 「ウィンドウ」や「画面全体」を選ぶと操作パネルが参加者に見えます。**唯一かつ最大の事故ポイント**です。
- パネル上部のランプが**赤（タブを一度クリックしてください）**のときは、まだ音が出ません。対象タブをクリックしてください。
- 対象タブで**別タブが開いた場合**（Googleスライドのプレゼン開始など）は、そのタブを選び直して接続します。同じタブ内で遷移した場合は自動で復帰します。
- 操作する人はヘッドホン推奨（スピーカーだとマイクに音が回り込むことがあります）。

---

## 使用例：社内表彰式（Google Meet）

司会進行・スライド担当が、進行しながら自分でBGMとドラムロールを出す構成。
BGM専任の担当者や2台目のPCは不要です。

- 事前に BGM（ループ）とドラムロール（効果音）を読み込み、Q1・Q2 に割り当てておく
- 本番はスライドを操作しながら `⌥⇧1` / `⌥⇧2` でキュー出し
- 表彰の合間にBGMを止めるときは `⌥⇧0`（フェードアウトで自然に消えます）

### リハーサル用チェックリスト

- [ ] 別の会議を立てて、参加者側の耳で音量を確認した
- [ ] BGMと話し声のバランスを確認した（BGMは会話の邪魔にならないか）
- [ ] ドラムロールを鳴らしてから発表するまでの間合いを通した
- [ ] BGMの停止（フェードアウト）が自然に聞こえるか確認した
- [ ] 全停止ボタン／`⌥⇧0` が効くことを確認した
- [ ] 本番と同じ「タブ共有 + タブの音声も共有」で確認した

---

## 制約

- 音源はブラウザのストレージに保存されます。Chromeのプロファイルを消すと消えます。
- ファイルは1本20MB程度までを推奨（それ以上は読み込みに時間がかかります）。
- 音が出ないときは、対象タブで開発者ツールのコンソールを開き `[TabAudioConsole]` のログを確認してください。
